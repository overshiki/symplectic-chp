# Unsupported RX Gate Test

## Circuit
```
RX 0
M 0
```

## Purpose

This circuit tests **error handling** for non-Clifford gates.

## Why RX is Unsupported

The RX gate is a rotation around the X-axis:
$$R_X(\theta) = e^{-i\theta X/2} = \cos(\theta/2)I - i\sin(\theta/2)X$$

For a generic angle, this is **not a Clifford gate**.

### Special Cases

1. **RX(π) = X** - Clifford (Pauli X)
2. **RX(π/2) = √X** - Clifford (if combined appropriately)
3. **RX(θ) for generic θ** - Non-Clifford

The STIM `RX` gate without parameters typically means RX(π) or a generic rotation, neither of which is directly supported by our restricted CHP simulator.

## CHP Limitations

The CHP simulator only supports the **Clifford group**:
- Generated by: H, S, CNOT
- Closed under: products, inverses
- Acts on Pauli group by conjugation

RX(θ) for θ ∉ {0, π/2, π, 3π/2, ...} takes Pauli operators to arbitrary rotations, not just other Pauli operators.

## Error Handling

The simulator should:
1. Detect the unsupported gate during translation
2. Report a clear error message
3. Exit gracefully

## Expected Behavior

```
Translation error: Unsupported gate: RX
This gate is not a Clifford gate or is not yet implemented.
```

## Note on Extension

To support RX, one would need:
1. **If RX = X**: Decompose as HSSH (already supported)
2. **If RX = √X**: Add native support or decomposition
3. **If generic RX**: Extend beyond CHP to full state vector simulation

This test ensures the simulator correctly rejects unsupported operations rather than silently producing incorrect results.
